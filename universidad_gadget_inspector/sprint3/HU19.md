### HU-019 — Clean Code: encapsulación y eliminación de código muerto

**Como** desarrolladora del equipo
**Quiero** que el código respete las convenciones de visibilidad de Python y no contenga ramas que nunca se ejecutan
**Para** que cualquier miembro del equipo pueda leer el código sin inferir comportamientos incorrectos por convenciones violadas

| # | Criterio de aceptación |
|---|---|
| AC-1 | **Given** que `_is_backlog(us)` se llama desde `run_validation.py` (fuera de la clase), **When** se renombra a `is_backlog(us)` eliminando el prefijo `_`, **Then** todos los sitios de llamada (`run_validation.py`, tests, regression) deben actualizarse **And** el linter no debe reportar acceso a métodos privados desde el exterior |
| AC-2 | **Given** que `if dynatrace_exporter:` es código muerto (el constructor lanza `ValueError` o retorna un objeto válido, nunca `None` ni falsy), **When** se elimina esa guardia, **Then** el `try/except` de Dynatrace debe quedar directo sin la condición previa **And** el comportamiento en tiempo de ejecución debe ser idéntico |
| AC-3 | **Given** que los 3 cambios se aplican simultáneamente, **When** se ejecuta la suite de tests en ese momento (105 tests), **Then** todos deben pasar sin necesidad de modificar ningún test de lógica **And** solo los tests de headers/mocks de nombre de método deben actualizarse para el rename |
