### HU-015 — Corrección de dimensiones Dynatrace con nombres de proyecto con espacios (B20)

**Como** analista de calidad que usa el dashboard de Dynatrace
**Quiero** que las métricas se envíen con los nombres de proyecto originales preservando espacios
**Para** que los filtros y tablas del dashboard funcionen correctamente y no muestren "no records" al filtrar por proyecto

| # | Criterio de aceptación |
|---|---|
| AC-1 | **Given** que un proyecto se llama `"CoE Calidad"` (con espacio), **When** `dynatrace_exporter.py` construye la línea MINT para el API de ingest, **Then** la dimensión debe ser `project="CoE Calidad"` con comillas preservando el espacio **And** no debe usar guión bajo ni ningún reemplazo del espacio |
| AC-2 | **Given** que el dashboard tiene la variable `$Proyectos` con valor `"CoE Calidad"`, **When** se consulta la tabla de métricas filtrando por ese proyecto, **Then** deben aparecer registros porque el nombre enviado coincide exactamente con la variable **And** la consulta no debe retornar vacío |
| AC-3 | **Given** que un valor de dimensión contiene backslashes (`\`) o comillas dobles (`"`), **When** `_format_series_line` formatea la dimensión, **Then** los `\` deben reemplazarse por `/` y las `"` deben eliminarse **And** la línea MINT resultante debe ser sintácticamente válida para el API `/api/v2/metrics/ingest` |
