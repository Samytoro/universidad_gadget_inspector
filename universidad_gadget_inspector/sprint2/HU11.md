### HU-011 — Tests E2E con Playwright para validación contextual de backlog (B9)

**Como** QA Engineer
**Quiero** tests end-to-end que verifiquen el flujo completo HTTP → FastAPI → validador → respuesta para HUs en backlog
**Para** garantizar que la integración entre capas funciona correctamente y no solo la lógica de negocio aislada

| # | Criterio de aceptación |
|---|---|
| AC-1 | **Given** que Playwright envía una HU con `state=New` y `assigned_to=null` al endpoint `/validar`, **When** recibe la respuesta, **Then** el status debe ser `200 OK` y `errors` debe ser `[]` **And** el test debe estar etiquetado `[B9]` para trazabilidad |
| AC-2 | **Given** que Playwright envía una HU con `iteration="Product Backlog"` y `state=Active` sin `assigned_to`, **When** recibe la respuesta, **Then** debe ser `200 OK` con `errors: []` **And** validar que la iteración "backlog" tiene precedencia sobre el estado activo |
| AC-3 | **Given** que Playwright envía una HU con `state=Active`, `iteration="Sprint 5"` y `assigned_to=null`, **When** recibe la respuesta, **Then** `errors` debe contener al menos un elemento **And** el test debe confirmar que las HUs activas siguen siendo rechazadas sin `assigned_to` |
| AC-4 | **Given** que existen 16 tests Playwright en total (11 previos + 5 nuevos de B9), **When** se ejecuta la suite completa con `npx playwright test`, **Then** deben pasar los 16 **And** el tiempo total de ejecución no debe superar 10 segundos |
